Procesos en Sistemas Operativos




Los procesos son las tareas que ejecuta el sistema de cómputo, por medio del sistema operativo, donde se prestan diferentes factores para que el desarrollo de esa tarea se realice, por ejemplo, el uso de memoria. Como complemento dice (La Red Martínez, 2004) “Aquello que se manifiesta por la existencia en el Sistema Operativo de un bloque de control de proceso”.
  • Eventos que provocan la creación de procesos.




    • Como parte de la ejecución de los procesos se establece según (Sol Llaven, 2016), los siguientes cuatro mecanismos.

    • Como parte del arranque del sistema operativo.
    • A partir de un proceso existente, mediante una llamada el sistema.
    • Con una petición del usuario mediante un intérprete de comandos o una interfaz gráfica.
    • Como parte del procesamiento por lotes en un sistema que lo realice de una manera automática.

  • Estados que conformado un proceso.

    • Se especifican tres estados en el inicio, desarrollo y fin de un proceso o tarea en los cuales presenta un ciclo que muestra la creación del proceso, ya sea por cualquiera de los eventos vistos anteriormente, el proceso de ejecución y el cierre, este último se caracteriza cuando ya no requiere atención del procesador. Como lo expone (Sol Llaven, 2016) “Cuando un proceso está preparado para ser atendido por el procesador se dice que está “listo” para la ejecución, cuando está haciendo atendido se dice que esta “bloqueado”.”

  • Condiciones para la terminación de un proceso




    • Se expone a continuación las cuatro condiciones para la terminación de un proceso son según lo escrito por (Sol Llaven, 2016):

    • Salida normal: Se ejecuta cuando el mismo programa a través de su algoritmo, da la orden de concluir el programa.
    • Por un error crítico: Sucede cuando el programa tiene que cerrar debido a alguna situación que le impide continuar con la operación, debido a esto, el programa ejecuta funciones que le permiten disminuir daños y recuperar información.
    • Por una condición de excepción: Se presenta cuando ante diversas situaciones inesperadas por el algoritmo, el proceso termina incluso si se encuentra ejecutando funciones de recuperación.
    • Recibir una señal de otro proceso: Cuando algunos procesos tienen la facultad de enviar señales a otros procesos, el comportamiento básico de es el de recibir señales y luego terminar el proceso.
